# Changelog — Lera2026 TDVRPTW BKS

All notable changes to the curated `Lera2026` TDVRPTW best-known solutions (BKS) are recorded here. Objective: **Duration** (duration minimization — the depot departure time of each route is a decision variable). Costs are the authoritative output of the canonical checker (`mamut_routing_lib.td.check_td_solution`): exact IEEE-754 double arithmetic, no epsilon thresholds, routes in canonical order (sorted by first customer), total summed in that order — so any strict improvement is real. Reminder: instances carry a minimal time-window repair relative to the raw Gehring & Homberger data (see README.md), so results are not comparable with static VRPTW literature.

## 2026-07-07 — family created

480 instances generated by `populate_td_lera2026` v1 (IGP 2003 Table 1 speeds, Dabia-2013 five-period skeleton, minimal TW repair, cross-validated against the exact forward Ichoua loop). Initial BKS seeding by kayros TD-ACO+LS under the paper6 protocol (per-size time limits 300/600/900/1200/1800 s, seeds {42, 123, 456}) on Grid'5000.
